PrintShop Personal Use

3D Printing Business Management & Profit Tracking

What is PrintShop?

PrintShop is a self-hosted business management tool built for Smartz Choices, a small 3D printing shop. It tracks material inventory (filament spools), product costs, marketplace orders, business expenses, and calculates real per-order profitability.

This is a personal tool for managing a single Etsy shop — it is not a third-party service, SaaS product, or publicly available application. Only the shop owner uses it.

Features

Dashboard & KPIs

Real-time P&L overview with revenue, COGS, fees, expenses, and net profit. Filterable by today, week, month, and year-to-date. Top products by profit and trend charts.

Filament Inventory

Track material batches by brand, type, and color. FIFO cost tracking, consumption logging, grams remaining per spool, low-inventory alerts, and weighted cost-per-gram calculations.

Product Catalog

Manage products with SKU variations, filament usage (grams), print time, packaging costs, and per-variation pricing. Link products to marketplace listings.

Order Management

View and manage orders across all sales channels. Per-order profit breakdown including COGS, marketplace fees, shipping, and discounts. Filter by date, channel, and status.

Marketplace Integrations

Pull orders automatically from connected marketplaces (eBay, Amazon). Manual order entry for in-person or direct sales. Unlinked order resolution for SKU matching.

Expenses & Cashflow

Track business expenses (equipment, supplies, subscriptions). Cashflow view showing money in vs. money out. Expenses factor into net profit calculations.

Reports

Business reports for analyzing profitability trends, product performance, and channel comparison over configurable time periods.

Multi-Channel Sales

Unified view across Etsy, eBay, Amazon, and manual sales. Per-channel revenue and profit breakdown on the dashboard.

Etsy API Integration

PrintShop integrates with the Etsy Open API v3 to automatically import orders from the shop owner's Etsy store. This eliminates manual data entry and enables accurate, real-time profit calculations.

How PrintShop Uses the Etsy API

  • Order sync — Automatically pull completed and in-progress orders from the connected Etsy shop at regular intervals (every 5 minutes).
  • Order details — Import order line items, quantities, pricing, shipping charges, and buyer information to calculate per-order profitability.
  • Fee tracking — Capture Etsy transaction fees, listing fees, offsite ads fees, shipping transaction fees, and payment processing fees via ledger entries for accurate net profit calculations.
  • Billing data — Read account-level charges (Etsy Plus subscription, monthly fees) to factor into overall business expense tracking.
  • Historical import — One-time bulk import of historical orders (last 7, 30, or 90 days) for backfilling data.
  • SKU matching — Match incoming Etsy order items to internal product catalog entries via SKU for automated COGS calculation.

Requested API Scopes

transactions_r listings_r billing_r shops_r
  • transactions_r — Read orders, receipts, payments, and payment account ledger entries (transaction fees, listing fees, offsite ads fees, shipping fees, disbursements).
  • listings_r — Read listing data to match Etsy listings to internal product catalog via SKU.
  • billing_r — Read account-level billing charges such as Etsy Plus subscription fees and other monthly charges.
  • shops_r — Read shop information for connection verification and shop identification.

All scopes are read-only. No write, update, or delete operations are performed on the Etsy shop, listings, or account.

Data Handling & Privacy

PrintShop is a self-hosted application running on a private server. All data — including Etsy API tokens, order data, and buyer information — is stored in a private PostgreSQL database accessible only to the shop owner. No data is shared with, sold to, or accessible by any third party. API credentials are never exposed or re-distributed.

  • OAuth 2.0 authentication with PKCE — tokens stored server-side only
  • No public-facing API — the application is accessed only by the shop owner
  • Buyer data (name, email) used solely for order identification, never for marketing
  • No data scraping — all data obtained through official Etsy API endpoints
  • API credentials stored in environment variables, never committed to source control

Technical Stack

Frontend: Next.js 14 (App Router, TypeScript)
Backend: NestJS (TypeScript, REST API)
Database: PostgreSQL with TypeORM
Styling: Tailwind CSS (dark theme)
Auth: Session-based + OAuth 2.0 for marketplaces
Hosting: Self-hosted via Docker (private server)
